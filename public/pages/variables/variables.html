Variables

We use variables to store data.
When we write variables in JS we have different keywords

const
let
var
.. nothing

BUT you should only use the keywords const and let

Const

  • ALWAYS use const - until it won't let you

2 rules when using const
1. You need to assign a value when you declare a variable

Examples:

// Correct way

const name = "Laura";


// Wrong way - will get an error

const name;

2. You cannot reassign

Examples:

const name = "Laura";


// Wrong - it will give you an error when you try to reassign a new value to 'name'

name = "Karen";

Note:
It is possible to change/modify an object who is stored in a const variable

const person = {
  firstName: "Laura",
  lastName: "Storm"
};


person.name = "Karen";

// output: { firstName: 'Karen', lastName: 'Storm' };


Let

  • Never use let - unless you have to
  • You use let when you have to reassign

// You can reassign like that

let sum = 33;

sum = 76;

Why is it bad?
Because you can modify/change a variable without noticing.


Never do global variables

  • Global variables is not good when you code, because they can be access from everywhere.
  • You can forget you wrote it and easily override/reassign them
  • var is a global variable

So never do this:

// Wrong to do - global variable

var globalVariable = "Never do this";


// Wrong to do - total global variable

totalGlobalVariable = "Also never do this";
