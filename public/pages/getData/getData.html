Query String & Path Variable

Query String and Path Variable are used to get data/information from the URL of an HTTP request

Query String

  • A Query String starts with a '?' and contains a key-value pair
  • If there are more than 1 key-value pair it will be seperated with a '&'
  • It is used to send data to the server for processing

Example
// URL
http://google.com/example?code=awesome&level=beginner

// app.js 

app.get("/example", (req, res) => {
    console.log(req.query.code) // output: awesome
    console.log(req.query.level) // output: beginner
    res.send({ message : `The value to the keys are ${req.query.code} and ${req.query.level} ` });
})


Notes:
http = protocol
google = domain
example = endpoint
?code=awesome&level=beginner = query string
code = key, awesome = value
level = key, beginner = value

Path Variable

  • Path Variable are mostly used to extract dynamic segments from the URL path
  • Example: this URL path '/birds/1' could be used to identify a specific bird with the ID 1.
    So 1 in the URL is the path variable. - and we extract that information to search in a database for example.
  • Path Variable allows us to create dynamic and flexible applications

Example
// URL
http://google.com/birds/1

// app.js 

app.get("/birds/:id", (req, res) => {
    console.log(req.params.id) // output: 1
    const foundBird = birds.find(bird => bird.id === Number(req.params.id));
    res.send({ data: foundBird });
})


Notes:
http = protocol
google = domain
/birds = endpoint
1 = path variable

'/birds/:id' = id is a placeholder for 1 in the URL. ':placeholder' is the syntax - and you can write whatever you want for the placeholder variable name.
req.params.id = is the way you extrac the path variable.
req = request
params = path variable
id = is where the path variable value is stored
