Node.js

  • Node JS is a sever site enviroment
    - a way to run JavaScript on your computer instead of running JavaScript in the browser
  • We use it to built a webserver
  • Use Node for a webserver that can recieve incoming request ('listen for requests') and then send a response to the request
  • With Node we can write front-end and backend in the same language (JavaScript)
  • When we use Node we can import modules/libraries (reuse-code) from existing packages from npm

  • We can console.log the application
  • Rest API (server)

package.json - Express

When we create/initialize a new project, we need to have package.json file

  • package.json descripes the dependencies in our project
  • package.json is a meta data file that descripes the project

3 things in package.json
  1. Scripts (Built-in)
  2. Define dependencies
  3. Meta information
Create package.json in terminal

1. Create package.json with express dependency

$ npm install express

Create a package.json file by 'Hand'

1. Create a package.json file

package.json

2. In package.json write express dependency 

{
    "dependencies": {
        "meaning-of-life": "1.0.0",
        "express": "4.18.2"
    }
}

3. To install dependencies write in the terminal

$ npm install

Commands

$ node -v

  • This command will tell you what version of Node you have installed
  • It is also a way to check if you have Node installed on your computer
Terminal command

$ node -v

// shows: v18.14.0

$ node

  • This command allows you to open Node in the terminal
  • You can execute JavaScript commands from the command line in the terminal
  • It will also tell you which version you have

    • OBS: to exit press control + c
Terminal command

$ node

// shows: Welcome to Node.js v18.14.0.

>

$ node filename.js

  • This command will run a JavaScript file
  • You write node and the path to the file you would like to run/execute
  • You can also stand in the right folder (using cd +ls) and write node filename.js

    • OBS: use 'tab' on your keyboard to finish your writing - to check that you are in the right folder
Terminal command

// run a script with path

$ node node/notes/myfile.js


// run a script from the right folder

$ node myfile.js

$ npm init || $ npm init -y

  • npm = Node Package Manager
  • init = initialize

  • Use this command npm init to create a new Node JS project.
    It will initialize a new package.json file

  • $ npm init -y = creates a package.json without you have to answer questions to the project
  • $ npm init = will ask you some questions and then create a package.json for you

    • OBS After we have created package.json we need to remember to install Express
Terminal command

// initialize project with questions

$ npm init


// initialize project without questions

$ npm init -y

$ npm install express

  • We install the Express module from npm, so we can use express in our project
  • If you haven't ran $ npm init and you run $ npm install express, it will create a package.json for you, with express dependency
Terminal command

$ npm install Express

$ npm install

  • We use npm install to run our dependencies from package.json
  • If you write dependencies in package.json remember to run this command after to 'update' the file
Terminal command

$ npm install

$ sudo npm install -g nodemon

  • Nodemon is a great developer tool to install
  • You don't have to write $ node app.js everytime you make a change in your code
  • It makes it easier for developers
  • When nodemon is installed globaly (-g) you write nodemon instead of node when you will run your application
Terminal command

$ sudo npm install -g nodemon

$ nodemon app.js

  • Use this command when you want to run the application (app.js)
  • It will automatically update everytime you make a change in the code
Terminal command

$ nodemon app.js
