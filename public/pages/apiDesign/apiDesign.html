REST API - Design

Before coding a REST API it is good practice to make a Design.
The Design should show the collections of endpoints in your application and what they will do.

When Design - focus on:

  1. Ordering of the HTTP methods
  2. The standardization of the endpoints which should reflect the underlying collections in your system
Words

collections = endpoint.
In mySQL it is tables


Design Example

HTTP Method Endpoint
GET /movies
GET /movies/< id >
POST /movies
PUT /movies/< id >
PATCH /movies/< id >
DELETE /movies/< id >
  • Methods: tells the HTTP request (GET, POST, PUT, PATHC, DELETE)
    Endpoints: shows endpoint
